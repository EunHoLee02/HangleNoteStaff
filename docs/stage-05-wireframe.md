# 5단계 텍스트 와이어프레임

## 화면명

악보 변환 작업 화면

## 레이아웃 구조

```text
[Header: Hangul Note Staff]                 [샘플] [악보 만들기]

[입력 패널]                                [악보 패널]
- 한글 음계 textarea                       - 오선보 미리보기 제목
- 시작 옥타브 / 박자                       - 요약 텍스트
- 기본 길이 / 빠르기                       - SVG 악보 영역
- PDF/TXT 올리기 / 지우기                  - SVG / PNG / MusicXML 다운로드
- 상태 메시지
```

## 상태

- 빈 상태: 중앙 안내 문구.
- 로딩 상태: 파일 추출 중 상태 메시지.
- 오류 상태: 좌측 상태 메시지를 error 스타일로 표시.
- 성공 상태: 악보 SVG와 요약 갱신.

## 주요 인터랙션

- 입력 후 `악보 만들기`를 누르면 파싱과 렌더링이 실행된다.
- 파일 업로드 완료 후 입력창이 추출된 음계로 교체되고 자동 렌더링된다.
- 다운로드 버튼은 현재 SVG/note 배열 기준으로 파일을 생성한다.

## 단계 종료 정리

1. 이번 단계 확정 사항: 실제 구현 화면 블록 확정.
2. 다음 단계 입력 핵심 요약: ERD/API는 없는 정적 구조, 파일 처리 보안 정책 중심.
3. 추가 확인 필요 항목: 모바일에서 다운로드 버튼 순서 선호.
4. 충돌 가능 항목: 서버 기반 PDF 처리 도입 시 화면에 업로드 진행률 필요.
