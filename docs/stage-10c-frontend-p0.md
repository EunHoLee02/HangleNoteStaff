# 10c단계 프론트엔드 P0 기능 구현

## 선행 충돌 점검

- 3단계 화면 설계와 5단계 와이어프레임은 단일 화면 구조로 일치.
- 7단계 기준 스택과 구현 파일 구조가 일치.
- 충돌 항목 없음.

## 구현 파일

- `index.html`: 입력, 설정, 업로드, 미리보기, 다운로드 UI.
- `styles.css`: 반응형 2열/1열 레이아웃과 상태 스타일.
- `src/app.js`: 이벤트 연결, 렌더링, 다운로드 처리.
- `src/scoreRenderer.js`: SVG 오선보 렌더링.

## 화면별 테스트 포인트

- 렌더링: 기본 샘플이 오선보로 표시된다.
- 상태 전환: 입력 없음 오류, PDF 로딩, 성공 메시지 표시.
- 입력 검증: 유효 계이름이 없으면 빈 악보 상태가 표시된다.
- API 실패 처리: PDF.js 로드 실패 시 오류 메시지 표시.
- 반응형: 860px 이하에서 1열 배치.

## 단계 종료 정리

1. 이번 단계 확정 사항: P0 프론트엔드 구현 완료.
2. 다음 단계 입력 핵심 요약: 정적 호스팅 배포와 로컬 실행 가이드 작성.
3. 추가 확인 필요 항목: 접근성 세부 검수.
4. 충돌 가능 항목: 외부 디자인 시스템 도입 시 CSS 재정리 필요.
