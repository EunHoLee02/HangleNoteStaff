# 12단계 테스트 및 검증 계획

## 반드시 테스트해야 하는 P0 기능

- 한글 음계 파싱: `도레미파솔라시도`, `도 레 미`, `쉼`.
- 음 길이 파싱: `/1`, `/2`, `/4`, `/8`, `/16`.
- 옥타브 파싱: `도'`, `도,`.
- SVG 오선보 렌더링.
- SVG/PNG/MusicXML 다운로드.
- TXT 업로드.
- 텍스트 레이어 PDF 업로드.

## 권장 테스트 시나리오

1. 기본 샘플을 열고 오선보가 표시되는지 확인한다.
2. `도레미파솔라시도`를 입력하고 8개 음표 요약이 표시되는지 확인한다.
3. `도/2 레/4 미/8 파/16` 입력 후 MusicXML을 다운로드한다.
4. 텍스트 파일 업로드 후 입력창이 갱신되는지 확인한다.
5. 스캔 PDF 업로드 시 제한 메시지가 표시되는지 확인한다.

## 출시 전 체크리스트

- 데스크톱/모바일 레이아웃 깨짐 없음.
- 유효하지 않은 입력에서 앱이 멈추지 않음.
- 다운로드 파일명이 올바름.
- MusicXML이 MuseScore 등에서 열림.
- PDF.js 실패 시 기본 입력 기능은 유지됨.

## 출시 차단 이슈 기준

- 기본 샘플 렌더링 실패.
- 다운로드 버튼 오류.
- 입력값이 SVG/XML에서 escape되지 않는 문제.
- PDF 실패가 전체 앱 오류로 번지는 문제.

## 단계 종료 정리

1. 이번 단계 확정 사항: P0 검증 범위와 출시 차단 기준.
2. 다음 단계 입력 핵심 요약: 수동 smoke test 후 배포 가능.
3. 추가 확인 필요 항목: 자동화 테스트 도입 여부.
4. 충돌 가능 항목: OCR 포함 시 테스트 범위 대폭 증가.
