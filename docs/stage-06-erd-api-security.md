# 6단계 ERD + API 명세 + 기술 반영형 보안 정책

## ERD 초안

P0는 서버 저장과 계정이 없으므로 DB 테이블 없음.

## API 명세 초안

P0는 정적 웹앱으로 API 없음.

## 필수 보안 정책

- 인증 정보 저장 방식: 인증 기능 없음.
- 비밀번호 암호화: 해당 없음.
- 세션/JWT: 해당 없음.
- 접근 제어: 모든 P0 기능 비로그인 사용.
- 입력값 검증: 파서는 허용된 음계 토큰만 note 배열로 변환하고 나머지 문자는 무시한다.
- 파일 업로드 보안: 파일은 브라우저 메모리에서만 처리하며 서버 전송 없음.
- PDF 처리: PDF.js로 텍스트 레이어만 읽는다. 스캔 PDF OCR은 P1.
- XSS 대응: 입력값은 SVG 텍스트 삽입 전 XML escape 처리한다.
- 로그 민감정보: 서버 로그 없음. 향후 분석 도입 시 원문 파일/입력 텍스트 수집 금지.

## 개발 시 주의사항

- CDN PDF.js가 실패해도 기본 입력/렌더링 기능은 유지한다.
- MusicXML 생성 시 사용자가 입력한 텍스트는 XML escape 처리한다.

## 단계 종료 정리

1. 이번 단계 확정 사항: DB/API 없음, 브라우저 내 파일 처리, escape 정책.
2. 다음 단계 입력 핵심 요약: 기술 스택은 정적 HTML/CSS/JS + 선택적 PDF.js.
3. 추가 확인 필요 항목: 완전 오프라인 PDF 추출 필요 여부.
4. 충돌 가능 항목: OCR 서버 도입 시 API/스토리지/보안 정책 재설계.
