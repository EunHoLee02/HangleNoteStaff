# 3단계 화면 설계 + 사용자 플로우

## 화면: 악보 변환 작업 화면

- 목적: 한글 음계를 입력하고 오선보와 다운로드 파일을 생성한다.
- 주요 사용자: 일반 사용자.
- 진입 조건: 브라우저에서 `index.html` 또는 로컬 서버 URL 접속.
- 주요 UI 구성 요소: 헤더, 음계 입력창, 설정 입력, PDF/TXT 업로드, 상태 메시지, 악보 미리보기, 다운로드 버튼.
- CTA: `악보 만들기`, `SVG`, `PNG`, `MusicXML`.
- 빈 상태: “한글 음계를 입력하면 악보가 표시됩니다.”
- 로딩 상태: PDF 파일 읽는 중 상태 메시지.
- 오류 상태: 유효 음표 없음, PDF 추출 실패.
- 성공 상태: 오선보와 다운로드 버튼 활성.

## 사용자 플로우

1. 사용자가 한글 음계를 입력하거나 PDF/TXT를 업로드한다.
2. 앱이 한글 계이름만 추출한다.
3. 사용자가 옥타브, 박자, 기본 길이, 템포를 조정한다.
4. `악보 만들기` 클릭 또는 업로드 완료 시 악보를 렌더링한다.
5. 사용자가 SVG/PNG/MusicXML 중 필요한 형식으로 다운로드한다.

## 반응형

- 데스크톱: 입력 패널과 악보 패널 2열.
- 모바일: 입력 패널, 악보 패널 순서의 1열.

## 단계 종료 정리

1. 이번 단계 확정 사항: 단일 작업 화면으로 P0 전체 제공.
2. 다음 단계 입력 핵심 요약: IA는 한 페이지 중심, 다운로드 액션은 같은 화면에 배치.
3. 추가 확인 필요 항목: 향후 예제 라이브러리 페이지 필요 여부.
4. 충돌 가능 항목: 관리자 화면 요구가 생기면 IA 변경 필요.
