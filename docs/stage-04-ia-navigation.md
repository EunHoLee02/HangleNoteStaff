# 4단계 IA 및 네비게이션 설계

## 전체 사이트맵

- `/`: 악보 변환 작업 화면.
- `/docs`: 로컬 문서 폴더, 앱 라우트 아님.

## 사용자용 네비게이션

- 상단: 서비스명, 샘플, 악보 만들기.
- 본문 좌측: 입력/설정/업로드.
- 본문 우측: 오선보 미리보기/다운로드.

## 관리자용 네비게이션

MVP에서는 관리자 기능 없음.

## URL 구조 초안

정적 웹앱 기준 `index.html` 단일 엔트리.

## 접근 권한 정책

모든 기능은 비로그인 사용 가능. 파일은 서버로 전송하지 않는다.

## 단계 종료 정리

1. 이번 단계 확정 사항: 단일 페이지 IA.
2. 다음 단계 입력 핵심 요약: 와이어프레임은 좌측 입력, 우측 미리보기 구조.
3. 추가 확인 필요 항목: 배포 시 별도 도움말 경로 필요 여부.
4. 충돌 가능 항목: 계정/저장 기능 추가 시 URL 및 권한 정책 변경.
