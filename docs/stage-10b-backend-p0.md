# 10b단계 백엔드 P0 기능 구현

## 선행 충돌 점검

- 2단계 P0 기능은 모두 브라우저 내 처리로 정의됨.
- 6단계 API/DB 없음 정책과 일치.
- 충돌 항목 없음.

## 구현 범위

백엔드 없음. P0 백엔드 기능은 프론트엔드 모듈로 대체한다.

## 생성 파일

- `src/parser.js`: 한글 음계 파싱.
- `src/pdfExtractor.js`: TXT/PDF 텍스트 추출.
- `src/musicXml.js`: MusicXML 생성.

## 테스트 케이스 목록

- `도레미파솔라시도`가 8개 note로 파싱된다.
- `도/2 레/4 미/8`의 duration이 반영된다.
- `쉼`이 rest note로 처리된다.
- 텍스트 PDF에서 추출한 음계가 입력창에 반영된다.

## 단계 종료 정리

1. 이번 단계 확정 사항: 백엔드 없이 P0 처리.
2. 다음 단계 입력 핵심 요약: 프론트엔드 화면과 렌더러 구현.
3. 추가 확인 필요 항목: 서버 OCR 도입 여부.
4. 충돌 가능 항목: 스캔 PDF까지 P0로 요구할 경우 백엔드 필요.
