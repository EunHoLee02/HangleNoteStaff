# 2단계 상세 기능 명세서

## 사용자 기능

| ID | 우선순위 | 기능명 | 입력값 | 처리 로직 | 출력값 | 예외 |
| --- | --- | --- | --- | --- | --- | --- |
| F-001 | P0 | 한글 음계 입력 | textarea string | `도/레/미/파/솔/라/시/쉼` 토큰 추출 | note 배열 | 유효 음표 없음 |
| F-002 | P0 | 붙여쓰기 파싱 | string | 공백 없이도 한 글자 단위 계이름 인식 | 정규화 음계 | 무시 가능한 일반 문자는 제외 |
| F-003 | P0 | A4 오선보 렌더링 | note 배열, 박자, 템포 | A4 세로 페이지에 12개 음표 단위 줄바꿈, 96개 음표 단위 페이지 분할 | A4 SVG 페이지 | 범위 밖 음은 보조선 표시 |
| F-004 | P0 | 이미지 다운로드 | SVG DOM | SVG 직렬화, canvas 변환, 여러 쪽이면 페이지별 파일 생성 | `.svg`, `.png` | 브라우저 canvas 실패 |
| F-005 | P0 | MusicXML 다운로드 | note 배열 | MusicXML 3.1 partwise 생성, 페이지 분할 위치에 print 힌트 삽입 | `.musicxml` | note 배열 없음 |
| F-006 | P0 | PDF/TXT 업로드 | file | TXT는 직접 읽기, PDF는 PDF.js 텍스트 레이어 추출 | 입력창 반영 | 스캔 PDF, PDF.js 로드 실패 |
| F-007 | P1 | OCR PDF | scanned PDF | OCR로 한글 음계 추출 | 입력창 반영 | OCR 정확도 낮음 |
| F-008 | P0 | PDF 다운로드 | A4 SVG 페이지 배열 | 페이지별 SVG를 이미지로 변환 후 A4 PDF 페이지로 삽입 | `.pdf` | PDF 라이브러리 로드 실패 |

## 상태값

- `READY`: 입력 대기.
- `RENDERED`: 악보 생성 완료.
- `ERROR`: 입력/파일 처리 실패.

## 권한 체크

MVP는 인증이 없으므로 권한 체크 없음. 파일은 브라우저에서만 처리한다.

## 로그/이벤트 포인트

서버 로그 없음. 향후 P2 분석을 붙일 경우 파일 원문은 수집하지 않는다.

## 관련 API / DB

P0 정적 웹앱이므로 API와 DB 없음.

## 단계 종료 정리

1. 이번 단계 확정 사항: F-001~F-006, F-008을 P0로 확정, F-007 OCR은 P1.
2. 다음 단계 입력 핵심 요약: 단일 화면에서 입력, 업로드, 미리보기, 다운로드를 제공한다.
3. 추가 확인 필요 항목: MusicXML 외 MIDI 내보내기 필요 여부.
4. 충돌 가능 항목: 서버 저장 기능을 추가하면 현재 개인정보/보안 정책 재설계 필요.
